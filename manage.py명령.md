## 🐍 `python manage.py` 뒤에 올 수 있는 주요 명령어들

`python manage.py`는 Django 프로젝트의 핵심 관리 유틸리티이며, 이 뒤에 붙는 명령어들은 Django 프로젝트를 개발, 운영, 관리하는 데 필요한 모든 작업을 수행합니다.

다음은 **Django 프로젝트를 다루는 IT 전문가로서 반드시 알아야 할** 가장 중요한 명령어들입니다.

-----

## 1\. ⚙️ 서버 및 개발 관련 명령어

| 명령어 | 역할 | 설명 |
| :--- | :--- | :--- |
| **`runserver`** | **개발용 웹 서버 실행** | Django 프로젝트를 로컬에서 테스트하기 위한 경량 웹 서버를 실행합니다. **운영 환경에서는 절대 사용하지 않습니다.** (예: `python manage.py runserver 0.0.0.0:8000`) |
| **`createsuperuser`** | **최고 관리자 계정 생성** | Django 관리자 사이트에 접속할 수 있는 권한을 가진 슈퍼유저 계정을 생성합니다. |
| **`startapp [앱 이름]`** | **새로운 애플리케이션 생성** | 프로젝트 내부에 특정 기능(예: `blog`, `accounts`)을 담당할 새로운 앱 폴더와 기본 파일을 생성합니다. |
| **`collectstatic`** | **정적 파일 수집** | 배포를 위해 여러 앱에 흩어져 있는 정적 파일(CSS, JS, 이미지)들을 한 곳(**`STATIC_ROOT`**)으로 모읍니다. **배포 직전에 필수적으로 실행**해야 합니다. |

## 2\. 💾 데이터베이스 및 마이그레이션 명령어

| 명령어 | 역할 | 설명 |
| :--- | :--- | :--- |
| **`makemigrations [앱 이름]`** | **마이그레이션 파일 생성** | `models.py` 파일의 변경 사항(새로운 모델, 필드 변경 등)을 감지하여 데이터베이스 변경 이력을 담은 `.py` 파일을 생성합니다. |
| **`migrate [앱 이름] [마이그레이션 이름]`** | **마이그레이션 적용** | `makemigrations`로 생성된 변경 이력 파일들을 읽어 **실제 데이터베이스에 반영**합니다. (가장 중요) |
| **`showmigrations`** | **마이그레이션 상태 확인** | 현재 프로젝트의 모든 앱과 마이그레이션 파일들의 **적용 여부**를 보여줍니다. |
| **`inspectdb`** | **기존 DB 스키마 리버스 엔지니어링** | **이미 존재하는 데이터베이스 테이블 구조**를 읽어 들여, 자동으로 `models.py`에 작성할 수 있는 **Django 모델 코드 형태**로 출력합니다. (레거시 DB를 Django로 전환할 때 매우 유용) |

## 3\. 🛡️ 테스트 및 관리 명령어

| 명령어 | 역할 | 설명 |
| :--- | :--- | :--- |
| **`test [앱 이름]`** | **단위 테스트 실행** | `tests.py` 파일에 정의된 테스트 케이스들을 실행하여 코드의 동작을 검증합니다. |
| **`shell`** | **Django 환경 Python 쉘 실행** | Django 환경이 로드된 상태의 Python 인터프리터에 접속합니다. 데이터베이스 모델을 직접 조작하고 테스트할 때 유용합니다. |
| **`check`** | **프로젝트 설정 및 문법 검사** | Django 설정 파일(`settings.py`)과 코드의 문법적 오류나 잠재적인 문제를 미리 검사합니다. |

## 4\. [실전 팁] Docker 환경에서의 활용

**3vm구축실습예제** 환경에서 `docker exec` 명령과 함께 `manage.py`를 사용하는 것은 일상적인 작업입니다.


**3vm구축실습예제** 환경에서 이미 존재하는 DB를 가져올 때, `inspectdb`를 사용하면 시간을 크게 절약할 수 있습니다.

```bash
# 1. 실행 중인 DB 컨테이너와 연결된 Django 앱 컨테이너 내부로 접속하여 명령 실행
# 2. 결과가 표준 출력(화면)으로 출력됨
docker exec [django_컨테이너명] python manage.py inspectdb

# 3. 모델 코드를 models.py 파일로 바로 저장 (리다이렉션 사용)
docker exec [django_컨테이너명] python manage.py inspectdb > myapp/models.py
```

이후 생성된 `models.py` 파일을 검토하고 수정하여 프로젝트에 활용하면 됩니다.


  * **배포 전 마이그레이션 적용 (필수):**

    ```bash
    docker exec [컨테이너명] python manage.py migrate
    ```

  * **배포 전 정적 파일 수집 (필수):**

    ```bash
    docker exec [컨테이너명] python manage.py collectstatic --noinput
    ```

  * **디버깅을 위한 쉘 접속:**

    ```bash
    docker exec -it [컨테이너명] python manage.py shell
    ```
