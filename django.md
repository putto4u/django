Django(장고)는 Python으로 작성된 **고수준(High-level)의 웹 프레임워크**입니다. 웹 개발을 빠르고 효율적으로 진행할 수 있도록 설계되었으며, 특히 복잡하고 데이터베이스 기반의 대규모 웹 애플리케이션을 구축하는 데 강력합니다.

---

## 1. 💡 Django의 특징과 철학

### A. Full-Stack Framework (풀 스택 프레임워크)
Django는 웹 개발에 필요한 거의 모든 기능(프론트엔드/백엔드)을 내장하고 있습니다. 개발자가 외부 라이브러리를 추가적으로 찾거나 설정할 필요가 거의 없도록 통합된 환경을 제공합니다.

### B. DRY (Don't Repeat Yourself)
**"반복하지 마라"**라는 원칙을 핵심 철학으로 삼고 있습니다. 코드의 중복을 줄여 유지보수성을 높이고 개발 시간을 단축시킵니다.

### C. Batteries Included (배터리 포함)
개발에 필요한 핵심 구성 요소들이 기본으로 제공됩니다.
* **ORM (Object-Relational Mapper):** Python 코드로 데이터베이스를 쉽게 조작할 수 있게 해줍니다.
* **관리자 인터페이스 (Admin Interface):** 코드를 거의 작성하지 않고도 데이터 관리를 위한 강력한 웹 관리 페이지를 자동으로 생성해 줍니다.
* **라우팅 및 템플릿 엔진:** URL 처리 및 동적 HTML 생성을 위한 도구가 내장되어 있습니다.

---

## 2. 🧩 Django의 구조: MTV 패턴

Django는 전통적인 MVC(Model-View-Controller) 패턴과 유사한 **MTV(Model-Template-View)** 패턴을 사용합니다. 

| 구성 요소 | 역할 | MVC 패턴에서의 대응 |
| :--- | :--- | :--- |
| **Model (모델)** | **데이터베이스 구조** 및 데이터 상호작용 정의. (ORM 사용) | Model (데이터) |
| **Template (템플릿)** | **사용자에게 보이는 화면** (HTML 구조). 데이터와 로직이 분리되도록 함. | View (화면) |
| **View (뷰)** | **비즈니스 로직 처리** 및 데이터 조작. 요청을 받아 모델을 통해 데이터를 처리하고 템플릿을 선택하여 응답을 반환. | Controller (제어) |

## 3. 🛠️ Flask와의 비교 (Expert Tip)

| 특징 | **Django** | **Flask** |
| :--- | :--- | :--- |
| **유형** | **풀 스택 프레임워크** | **마이크로 프레임워크** |
| **규모/목적** | 대규모, 복잡한 프로젝트, 데이터베이스 의존적인 애플리케이션 (예: 블로그, CMS, 소셜 네트워크) | 소규모, 간단한 API 서버, 특정 기능만 구현하는 서비스 |
| **내장 기능** | ORM, 관리자 페이지, 인증 시스템 등 **대부분 내장** | 핵심 기능만 제공, **나머지는 확장(Extension) 필요** |

Django는 개발자가 표준화된 방식으로 빠르게 대형 웹 서비스를 구축해야 할 때 최적의 선택입니다.
